# Домашнее задание к занятию 1.  «Введение в виртуализацию»
## Задача 2
### высоконагруженная база данных MySql, критичная к отказу
Задача не вполне четко сформулирована. Для данной задачи скорее всего подойдет платформа аппаратной виртуализации, поддерживающая фермы высокой доступности, например VMWare vSphere. Так же VMWare дает широкие возможности работы с устройствами ввода-вывода.
Для данной задачи можно было бы рассмотреть и физические сервера, так как это исключило бы точку отказа в виде гипервизора виртуализации и обеспечило бы самую высокую производительность за счет исключения оверхеда на виртуализацию и по процессору и по вводу-выводу.
### различные web-приложения
Виртуализация уровня ОС. Так как веб приложения обычно легко масштабируются/балансируются и резервируются, то для данной задачи максимально подойдут контейнеры, например docker.
### Windows-системы для использования бухгалтерским отделом
Аппаратная виртуализация. Конечные пользователи обычно привязаны к привычному интерфейсу, а так же ограничены поддерживаемыми их прикладным ПО ОС. Аппаратная виртуализация позволит развернуть любую версию ОС и установить на нее необходимый софт, а пользователям предоставит привычный интерфейс как на ПК. Так же можно было бы рассмотреть решение от Citrix.
### системы, выполняющие высокопроизводительные расчёты на GPU
Физические сервера. Не всякая платформа виртуализации предоставляет доступ к устройствам GPU. Кроме того, нет смысла дробить ресурсы гипервизором и тратиться на накладные расходы от виртуализации, когда приложение требует монолитной высокой вычислительной мощности

## Задача 3
### 100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based-инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий.
Для данной задачи подойдет виртуализация на базе Microsoft Hyper-V или VMWare vSphere, которые предоставляют инструменты для репликации данных и балансировки, а так же интегрируются с различными системами резервного копирования.

### Требуется наиболее производительное бесплатное open source-решение для виртуализации небольшой (20-30 серверов) инфраструктуры на базе Linux и Windows виртуальных машин.
Исходя из материалов курса, наиболее производительным из бесплатных гипервизоров будет Xen. Так же можно рассмотреть KVM и  VMWare ESXi  

### Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows-инфраструктуры.
Максимально совместимым для виртуализации Windows видимо будет Microsoft Hyper-V, однако сомневаюсь, что его можно назвать бесплатным. Вероятно можно посмотреть в сторону KVM и Xen, которые будут и производительны, и совместимы с большинством серверов, на которых будет развернтыа виртуализация. 

### Необходимо рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux.
Если имеется в виду необходимость развертывания полноценных виртуальных машин под управлением различных дистрибутивов Linux, то оптимально было бы воспользоваться KVM. Если же речь не идет о полноценных виртуальных машинах, а достаточно виртуализировать сервисы, то вероятно для этого будет достаточно docker.

## Задача 4
### Проблемы:
- необходимо много разных компетенций для управления различными системами
- сложности при управлении мощностями из-за их раздробленности
- фрагментация вычислительных ресурсов и как результат снижение эффективности их использования
- возможные проблемы в коммуникациях между сервисами, развернутыми на различных системах виртуализации
- сложности при переносе сервисов из одной системы виртуализации в другую

### Стал бы я использовать гетерогенную систему?
Для каждой задачи существует лучший инструмент, поэтому в рамках одного проекта вполне могут быть использованы как физические серверы, так и виртуальные машины, а так же сервисы на базе контейнеров.
Иногда из-за требований высокой доступности и отказоустойчивости, например, может так же потребоваться использовать не только ресурсы частного облака или фермы виртуализации, но и облачных провайдеров.
