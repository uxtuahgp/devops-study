# Домашнее задание к занятию 1.  «Введение в виртуализацию»
## Задача 2
### высоконагруженная база данных MySql, критичная к отказу
Задача не вполне четко сформулирована. Для данной задачи подойдет платформа паравиртуализации, поддерживающая фермы высокой доступности, например VMWare vSphere. Так же VMWare дает широкие возможности работы с устройствами ввода-вывода.
Для данной задачи можно было бы рассмотреть и физические сервера, так как это исключило бы точку отказа в виде гипервизора виртуализации и обеспечило бы самую высокую производительность за счет исключения оверхеда на виртуализацию и по процессору и по вводу-выводу.
### различные web-приложения
Виртуализация уровня ОС. Так как веб приложения обычно легко масштабируются/балансируются и резервируются на уровне приложения, то для данной задачи лучше выбрать наиболее дешевую, простую в развертывании и управлении платформу.
### Windows-системы для использования бухгалтерским отделом
Паравиртуализация. Конечные пользователи обычно привязаны к привычному интерфейсу, а так же ограничены поддерживаемыми их прикладным ПО ОС. Паравиртуализация позволит развернуть любую версию ОС и установить на нее необходимый софт, а пользователям предоставит привычный интерфейс как на ПК.
### системы, выполняющие высокопроизводительные расчёты на GPU
Физические сервера. Не всякая платформа виртуализации предоставляет доступ к устройствам GPU. Кроме того, нет смысла дробить ресурсы гипервизором и тратиться на накладные расходы от виртуализации, когда приложение требует монолитной высокой вычислительной мощности

## Задача 3
### 100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based-инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий.
Для данной задачи подойдет паравиртуализация на базе Microsoft Hyper-V или VMWare vSphere, которые предоставляют инструменты для репликации данных и балансировки, а так же интегрируются с различными системами резервного копирования.

### Требуется наиболее производительное бесплатное open source-решение для виртуализации небольшой (20-30 серверов) инфраструктуры на базе Linux и Windows виртуальных машин.
Исходя из материалов курса, наиболее производительным из бесплатных гипервизоров будет Xen. Так же можно рассмотреть KVM и  VMWare ESXi  

### Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows-инфраструктуры.
Максимально совместимым для виртуализации Windows видимо будет Microsoft Hyper-V, однако сомневаюсь, что его можно назвать бесплатным. Вероятно можно посмотреть в сторону KVM и Xen, которые будут и производительны, и совместимы с большинством серверов. 
### Необходимо рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux.
Если имеется в виду необходимость развертывания полноценных виртуальных машин под управлением различных дистрибутивов Linux, то оптимально было бы воспользоваться KVM. Если же речь не идет о полноценных виртуальных машинах, а достаточно виртуализировать сервисы, то вероятно для этого будет достаточно docker.
